# 구글 설문지 템플릿 가이드

## 🚀 자동 생성 기능 사용 (권장)

가장 쉬운 방법은 `createFormAutomatically()` 함수를 사용하는 것입니다:

1. Apps Script 편집기에서 `createFormAutomatically` 함수 선택
2. 실행 버튼 클릭
3. 실행 로그에서 생성된 설문지 URL과 ID 확인
4. `Code.gs` 파일의 `FORM_ID`를 생성된 ID로 업데이트

이 함수는 참가자 명단을 기반으로 자동으로 설문지를 생성합니다.

## 📋 설문지 구조 (자동 생성 시)

자동 생성된 설문지는 다음 구조를 가집니다:

### 1. 설명
- 설문지 상단에 설명 텍스트 표시
- 채점 항목 구조 및 평가 방법 안내
- 창의성(30점), 구현 완성도(30점), 필요성(30점), 발표 전달력(10점) 총 100점 만점

### 2. 이름 선택
- **유형**: 드롭다운 (목록)
- **옵션**: 참가자 명단의 모든 이름
- **필수**: 예

### 3. 팀 선택
- **유형**: 드롭다운 (목록)
- **옵션**: 참가자 명단의 모든 팀명
- **필수**: 예

### 4. 각 팀별 평가 섹션
각 팀마다 다음이 생성됩니다:

#### 4-1. 섹션 헤더
- 팀명 표시 (예: "팀A 평가")
- 안내 문구: "각 항목을 1-10점 사이로 평가해주세요. (총 100점 만점)"

#### 4-2. 채점 항목 (4개 카테고리, 총 10개 항목)

각 팀은 다음 카테고리별로 평가됩니다:

**1. 창의성 (30점)**
- 아이디어와 결과물의 창의성 (10점 만점)
- 실행 가능성과 독창성 (10점 만점)
- 차별성 (10점 만점)

**2. 구현 완성도 (30점)**
- 필수 기능 구현 (10점 만점)
- 오류 처리 (10점 만점)
- 기술 이해도 (10점 만점)

**3. 필요성 (30점)**
- 확장성 (10점 만점)
- 주제 적합성 (10점 만점)
- 상품성 (10점 만점)

**4. 발표 전달력 (10점)**
- 발표 속도와 표현 (5점 만점)
- 발표자료 구성 (5점 만점)

**설정**:
- 모든 항목: 척도(Scale) 형식
- 최소값: 1
- 최대값: 10
- 라벨: "최저" / "최고"
- 필수: 예
- 각 항목의 실제 점수는 시스템이 자동으로 변환 (예: 5점 만점 항목에서 8점 입력 = 4점)

#### 4-3. MVP 점수
- **제목**: "[팀명] - MVP 점수"
- **유형**: 척도(Scale)
- **범위**: 1-10점
- **필수**: 예
- **안내**: "해당 팀의 MVP 점수를 평가해주세요."

**중요**: 본인 팀은 설문지에서 자동으로 제외됩니다.

## 📝 수동 생성 시 주의사항

수동으로 설문지를 만들 경우 다음 순서를 정확히 따라야 합니다:

1. **설명** 추가
2. **이름 선택** (드롭다운)
3. **팀 선택** (드롭다운)
4. **각 팀별로** (모든 팀 포함, 본인 팀은 응답 처리 시 자동 제외):
   - **창의성 섹션** (30점)
     - 아이디어와 결과물의 창의성 (10점 만점)
     - 실행 가능성과 독창성 (10점 만점)
     - 차별성 (10점 만점)
   - **구현 완성도 섹션** (30점)
     - 필수 기능 구현 (10점 만점)
     - 오류 처리 (10점 만점)
     - 기술 이해도 (10점 만점)
   - **필요성 섹션** (30점)
     - 확장성 (10점 만점)
     - 주제 적합성 (10점 만점)
     - 상품성 (10점 만점)
   - **발표 전달력 섹션** (10점)
     - 발표 속도와 표현 (5점 만점)
     - 발표자료 구성 (5점 만점)
   - **MVP 점수** (10점 만점)

## 🔄 설문지 업데이트

이미 설문지가 있고 팀이 추가된 경우:

1. Apps Script 편집기에서 `updateExistingForm` 함수 선택
2. 실행 버튼 클릭
3. 새로운 팀에 대한 평가 항목이 자동으로 추가됩니다

## ⚠️ 주의사항

1. **질문 순서**: 반드시 위 순서를 따라야 합니다
2. **팀명 일치**: 설문지의 팀명은 참가자 명단의 팀명과 정확히 일치해야 합니다
3. **이름 일치**: 설문지의 이름 옵션은 참가자 명단의 이름과 정확히 일치해야 합니다
4. **본인 팀 제외**: 설문 응답 처리 시 본인 팀 평가는 자동으로 제외됩니다

## 📊 응답 처리

설문 응답은 자동으로 처리됩니다:
- 각 팀의 10개 항목 점수가 각 항목의 최대 점수에 맞춰 변환되어 합산
- 총점 계산: 창의성(30점) + 구현 완성도(30점) + 필요성(30점) + 발표 전달력(10점) = 100점
- MVP 점수는 별도로 저장 및 계산
- 본인 팀 평가는 자동으로 제외됨

**점수 변환 예시**:
- 10점 만점 항목에서 8점 입력 → 8점
- 5점 만점 항목에서 8점 입력 → 4점 (8/10 × 5)
